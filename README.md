## 🛠️ Development Setup

### Prerequisites
- Node.js v18+
- npm or yarn
- Git
- Foundry (for smart contract compilation)

### Installation

1. **Clone the repository**:
```bash
git clone https://github.com/ContractForgeEVM/contractforge.git
cd contractforge
```

2. **Install dependencies**:
```bash
# Frontend
cd smart-contract-deployer
npm install

# Backend API
cd ../smart-contract-compiler-api
npm install
```

3. **Environment Configuration**:

Create `.env` files in each directory:

**smart-contract-deployer/.env**:
```env
VITE_INFURA_PROJECT_ID=your_infura_project_id
VITE_ALCHEMY_API_KEY=your_alchemy_api_key
VITE_WALLETCONNECT_PROJECT_ID=your_walletconnect_project_id
VITE_API_URL=http://localhost:3004
VITE_SUPABASE_URL=your_supabase_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
```

**smart-contract-compiler-api/.env**:
```env
NODE_ENV=development
PORT=3004
SUPABASE_URL=your_supabase_url
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_key
FOUNDRY_PATH=/usr/local/bin/forge
ETHERSCAN_API_KEY=your_etherscan_api_key
```

4. **Important Security Note**:
   
   ⚠️ **Protected Files**: The following critical files are protected and not available on GitHub for security reasons:
   - `smart-contract-compiler-api/src/utils/contractGenerator.ts` (Backend generator)
   - `smart-contract-deployer/src/utils/contractGenerator.ts` (Frontend generator)
   - `deploy-*.sh` (Deployment scripts)
   - `debug-*.js` (Debug scripts)
   - `test-*.js` (Test scripts)

   These files are maintained locally only and contain sensitive logic for contract generation and deployment.

5. **Start development servers**:

```bash
# Terminal 1 - Frontend
cd smart-contract-deployer
npm run dev

# Terminal 2 - Backend API
cd smart-contract-compiler-api
npm run dev
```

### Development Guidelines

- **Contract Generators**: The core contract generation logic is protected and maintained separately
- **Premium Features**: All 44+ premium features are fully implemented with 97.50% success rate
- **Template Synchronization**: Frontend and backend generators are synchronized
- **Security First**: Sensitive files are protected by comprehensive .gitignore rules
